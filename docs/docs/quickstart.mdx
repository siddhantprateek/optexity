This guide walks you through creating a simple login automation. By the end, you'll understand how to define actions, use parameters, and run your first Optexity workflow.

## Prerequisites

Make sure you've completed the [Installation](/docs/installation) guide and have Optexity set up in your environment.

## Your First Automation

Let's create an automation that logs into a website. We'll use a simple example that:

1. Clicks a "Sign in" link
2. Enters an email address
3. Enters a password
4. Clicks the "Sign In" button

### Step 1: Import the Required Components

```python
from optexity.schema.actions.interaction_action import (
    ClickElementAction,
    InputTextAction,
    InteractionAction,
)
from optexity.schema.automation import ActionNode, Automation, Parameters
```

### Step 2: Define Your Automation

```python
login_automation = Automation(
    url="https://example.com",
    parameters=Parameters(
        input_parameters={
            "username": ["user@example.com"],
            "password": ["mypassword123"],
        },
        generated_parameters={},
    ),
    nodes=[
        # Step 1: Click the Sign in link
        ActionNode(
            interaction_action=InteractionAction(
                click_element=ClickElementAction(
                    command="""get_by_role("link", name="Sign in")""",
                    prompt_instructions="Click the Sign in link",
                )
            )
        ),
        # Step 2: Enter the email
        ActionNode(
            interaction_action=InteractionAction(
                input_text=InputTextAction(
                    command="""get_by_role("textbox", name="Email")""",
                    input_text="{username[0]}",
                    prompt_instructions="Enter the email address",
                )
            )
        ),
        # Step 3: Enter the password
        ActionNode(
            interaction_action=InteractionAction(
                input_text=InputTextAction(
                    command="""get_by_role("textbox", name="Password")""",
                    input_text="{password[0]}",
                    prompt_instructions="Enter the password",
                )
            )
        ),
        # Step 4: Click Sign In
        ActionNode(
            interaction_action=InteractionAction(
                click_element=ClickElementAction(
                    command="""get_by_role("button", name="Sign In")""",
                    prompt_instructions="Click the Sign In button",
                )
            )
        ),
    ],
)
```

## Understanding the Structure

### Automation

The `Automation` object is the top-level container that defines:

- **`url`**: The starting URL for your automation
- **`parameters`**: Input values and generated values used during execution
- **`nodes`**: A list of actions to execute in sequence

### Parameters

Parameters are divided into two types:

- **`input_parameters`**: Values you provide upfront (like credentials, search terms, etc.)
- **`generated_parameters`**: Values extracted during automation (like order IDs, confirmation numbers)

Values are stored as lists of strings. Access them using `{variable_name[index]}` syntax:

```python
input_parameters={
    "username": ["user@example.com"],  # Access as {username[0]}
    "password": ["secret123"],          # Access as {password[0]}
}
```

### ActionNode

Each `ActionNode` represents a single step in your automation. An ActionNode can contain exactly one of:

- `interaction_action` - Click, type, select, scroll, navigate
- `extraction_action` - Extract data from the page
- `assertion_action` - Verify conditions
- `python_script_action` - Run custom Python code
- `fetch_2fa_action` - Handle two-factor authentication

### Locating Elements

Optexity uses Playwright locators to find elements. The `command` field accepts Playwright locator syntax:

```python
# Find by role
command="""get_by_role("button", name="Submit")"""

# Find by text
command="""get_by_text("Welcome")"""

# Find by test ID
command="""get_by_test_id("login-button")"""

# Find by CSS selector
command="""locator("#email-input")"""
```

The `prompt_instructions` field provides a natural language description that helps the LLM find the element if the locator fails.

## Running Your Automation

Once you've defined your automation, you can run it through the inference API:

```bash
curl -X POST http://localhost:9000/inference \
  -H "Content-Type: application/json" \
  -d '{
    "endpoint_name": "login-flow",
    "input_parameters": {
      "username": ["user@example.com"],
      "password": ["mypassword123"]
    },
    "unique_parameter_names": ["username"]
  }'
```
